== STEP 1 - SEND TO INTENT AGENT ==

You must build the message with an anti-repetition preamble. Use this EXACT format:

  "IGNORE ALL PREVIOUS REQUIREMENTS. THIS IS A NEW REQUEST.\n"
  + "REQUEST_ID: " + current_date_and_time + "\n"
  + "---REQUIREMENT_START---\n"
  + the user requirement text
  + "\n---REQUIREMENT_END---\n"
  + "PROCESS ONLY THE REQUIREMENT ABOVE. DO NOT REUSE ANY PREVIOUS RESPONSE."

For REQUEST_ID, use the current date/time in any format (e.g. "2026-04-08T21:10").

Call send_message_to_agents_matching_tags with:
  - match_all = ["creviz-intent-agent"]
  - match_some = ["creviz-intent-agent"]
  - message = the full string you built above

CRITICAL: You MUST include the preamble "IGNORE ALL PREVIOUS REQUIREMENTS" at
the start AND the suffix "DO NOT REUSE ANY PREVIOUS RESPONSE" at the end.
This prevents the Intent Agent from replaying a stale cached answer.

== STEP 2 - EXTRACT INTENT AGENT RESPONSE ==

The tool send_message_to_agents_matching_tags returns output like this:

  [{'agent_id': 'agent-xxxx', 'response': ['{ ... JSON string ... }']}]

This is a LIST containing a DICT with a 'response' key whose value is an ARRAY.

TO EXTRACT THE JSON PAYLOAD:
  1. Take the first element of the outer list (the dict)
  2. Get the 'response' array from that dict
  3. Take the first element of the 'response' array — THIS is the JSON string
  4. The JSON string starts with { and ends with }

IMPORTANT WARNINGS:
- The tool call parameters (which contain your original ---REQUIREMENT_START---
  message) are NOT part of the response. IGNORE them completely.
- Only extract from the 'response' array inside the returned dict.
- Do NOT include the agent_id, the list brackets, or the dict wrapper.
- Do NOT include any ---REQUIREMENT_START--- or ---REQUIREMENT_END--- text.
  Those are your OWN outbound message, not the Intent Agent's reply.

DOUBLE-ENCODING UNWRAP:
If the extracted string starts with " (double-quote) and ends with " (double-quote)
instead of { and }, it is double-encoded:
  Step A: Remove the first and last " characters
  Step B: Replace every \\" with "
  Step C: Result should now start with { and end with }

VALIDATION:
The extracted JSON must contain these keys: "mode", "requirement", "intents_detected", "instruction"
If it does NOT start with { or is missing these keys, tell the user:
  "ERROR: Intent Agent did not return a valid JSON payload. Please try again."

IMPORTANT: After extracting the JSON payload from the Intent Agent, check the "mode" field.

```json
<paste the COMPLETE RAW JSON from the "content" field of the tool output here>
```

CRITICAL: You MUST paste the ENTIRE raw JSON array from the "content" field.
DO NOT summarize, paraphrase, or reformat the JSON into bullet points.

The user needs the exact JSON structure, not a human-readable summary.
